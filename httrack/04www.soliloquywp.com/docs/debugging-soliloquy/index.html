Debugging Soliloquy

Table of Contents

WordPress has tons of plugins and themes, and because of the variety of choices, there is no way to shelter Soliloquy from conflicts completely. Conflicts are bound to happen, and I’ve been fortunate to work through a bunch of them.

I’ve listed below some of the most common bugs that occur with Soliloquy and some potential resolutions for each. I encourage you to scour through this list before submitting a support ticket.

**Quick Tip** Use Ctrl + F (or Cmd + F on a Mac) to search for terms on this page that are related to your issue.

GETTING STARTED

First things first. When debugging Soliloquy, I would highly recommend using Google Chrome. They have a very helpful debugging tool that lets you know exactly where errors are occurring and the nature of them. When in Google Chrome, do the following:

  1. Visit the webpage where the error is occurring.
  2. Right click on the page and select the “Inspect Element” option from the screen popup.
  3. Once the debugging window has appeared, click on the “Console” tab to the right.

If there are any JavaScript errors on the page, they will be listed in this console area. The messages output can be very beneficial for debugging Soliloquy.

Always make a backup of your site before debugging in case you accidentally delete or remove something that you shouldn’t!

ADMIN DEBUGGING

If you are having issues in the admin area of Soliloquy, here are some possible conflicts + resolutions for removing the conflicts.

#1: Help! I can’t upload any images – the upload images button doesn’t work!

I’m 99% sure that this is a conflict with either your theme or another plugin. The error is generated by invalid JavaScript that has polluted the Soliloquy add/edit screen, likely by another plugin or your own theme. Use the Google Chrome debugging method above to see if any JS errors are being output to the console.

Resolution

  1. Deactivate/reactivate your plugins one by one to resolve conflicts. Refresh the add/edit screen after each deactivation to determine the plugin that is conflicting with Soliloquy and contact their support to resolve the issue.
  2. Flush out your browser cache. Sometimes browsers generate invalid cache pages that do not serve JavaScript correctly.
  3. If neither of these work, try switching themes

#2: My image metadata won’t save!

See above. The issue is likely stemming from a conflict with another plugin or your theme.

Resolution

  1. See resolution for admin issue #1.

#3: My license won’t work and says it is already being used! Help!

Once in a blue moon, license keys are assigned incorrectly. If you receive this message and have not already attempted to activate your license elsewhere…

Resolution

  1. Login to your Member’s Area dashboard and find the license key on the right hand side of the page.
  2. Click on the orange “Deactivate” button to deactivate the license from any existing sites.
  3. Return to the license settings page and enter your license to have it verified.

#4: I’ve tried to deactivate my single license from a site, but it says there is an error.

If you are having issues with deactivating your single site license from a site and have already attempted to deactivate it from your Member’s Area dashboard (using the resolution steps from #3 above), you can do the following to reset your license on the site:

Resolution

  1. Add the following code right after the opening <?php tag of your theme’s functions.php file: delete_option( 'soliloquy_license_key' );
  2. Save the file and reload a page in the admin.
  3. Remove the code from the file and save the file. Your license data has been reset on the site.

#5: I don’t see the Addons menu anywhere?

In order for the Addons menu to be present, you need to have verified a developer license on the site.

Resolution

  1. Head to the Settings page and verify your developer license.
  2. If you do not have a developer license, either purchase or upgrade to a developer license from the Soliloquy website.

#6: You’ve said that there is an update for Soliloquy, but I don’t see it reflected in the site Updates area.

Unless there is a technical glitch on my end, you just need to refresh the Updates page a couple of times to see the update active. Refreshing the Updates page causes WordPress to check for updates, and Soliloquy will be added into those updates if there is indeed an update to the plugin (or Addons).

OUTPUT DEBUGGING

If you are having issues with Soliloquy displaying on the front end, check out the following items and see if a resolution is possible. Remember, please use Google Chrome to asses if there are any JavaScript errors output in the console.

#1: [Object object] has no method ‘soliloquy’.

This error comes from an issue with loading the Soliloquy script on the page, likely due to a previous script that has loaded in a bug. Check to see if any errors have occurred before it so you can isolate and deal with the bug.

Resolution

  1. Check to see if there are any other errors before this one and see if it is indicative of another plugin you are using. Try deactivating your plugins to see if the issue gets resolved, and then contact the support of the buggy plugin to resolve the issue.
  2. Clear out your browser cache (especially if the error appears after upgrading Soliloquy). Make you clear it out from the beginning of time in Chrome to ensure that no invalid cache responses remain.

#2: Help – Soliloquy won’t load on my page!

This is due likely to a JS error caused by your theme or a third party plugin.

Resolution

  1. Make sure that there is only one copy of jQuery being loaded on your site. If more than one copy of jQuery is being loaded, it will overwrite previous jQuery objects and discard anything attached to the previous. It will also slow down your site, so you should remove it anyway.
  2. Check your Chrome console for errors. There is likely a JS syntax or parsing error somewhere on the page. Try to resolve the issue by figuring out what script is causing the issue and where it is loaded from (either your theme or another plugin).
  3. Make sure that your theme has this somewhere in the footer.php file: wp_footer();

#3: The loading icon gets stuck and the slider doesn’t appear.

This means that there was an error loading the Soliloquy script.

Resolution

  1. Clear out your browser cache. It could be serving an invalid cache file. Make sure to clear it out to the earliest date possible (from the beginning of time in Chrome). Also, flush your plugin and CDN caches if applicable.
  2. Try deactivating plugins to see if one of the plugins is causing Soliloquy not to be able to load.

#4: The slider is not displaying like it should (it is off center, the navigation bullets are off, etc.).

If there is a display issue with the slider, it is likely because of a CSS conflict.

Resolution

  1. Open up the site in Google Chrome. Right click on the element that is not displaying correctly and hit “Inspect Element”. Highlight the element with the incorrect display and view the CSS to see if your theme is overwriting Soliloquy styles. You can test edit the CSS right from the debugging console to see what clears up the issue, and then you can apply fixes to the style.css file of your theme (or wherever you store your CSS customizations). 
  2. Make sure to view the HTML Structure of Soliloquy and make use of the !important CSS declaration if necessary to overwrite any styling issues.

#5: My slide transitions are jerky – help!

The is likely due to a CSS conflict with styling the <li> elements in the slider.

Resolution

  1. Try adding this to your theme’s style.css file: .soliloquy-slides, .soliloquy-item { margin: 0 !important; padding: 0 !important; }

#6: I need to center or float my slider.

See this article on positioning your slider: http://soliloquywp.com/docs/positioning-your-slider/
