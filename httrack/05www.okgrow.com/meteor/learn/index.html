Why learn Meteor?

So you want to learn how to build modern web apps. Today's minimum baseline is no longer pages generated on the server and forms with submit buttons. It's a browser-based JavaScript app. Backbone showed us what was possible and gave us a new way to look at web development. AngularJS and Ember took it to the next level.

But these frameworks are all missing an important part of a modern app: realtime data and live page updates. We now expect the page to update in response to new data, not just when we navigate or manually refresh the page.

You can do this the painful way and waste your time: build an AngularJS app for the front-end, then build a back-end app in Rails or Node + Express, or whatever you like. Now hook them up with a JSON data API, add a websocket connection for real-time updates. Phew, this is sounding like a lot of work already. Here’s what the Trello team went through back when there was no framework that could provide all of this.

We don't like to waste time and effort. We always choose the tool that gets us the furthest with the least work. The proverbial bicycle for the mind.

Meteor is a revolution just like Rails was 10 years ago - a huge leap in development efficiency. Meteor gives you a modern web app in a fraction of the time, and gives you live data and page updates for free.

Meteor builds on the best tools available:

  • Node.js, a proven back-end system
  • MongoDB
  • The websocket standard for realtime data
  • And of course it’s pure JavaScript, seamlessly running on both the server and the client.

Meteor is a great open-source project with an all-star development team and a ton of funding to keep improving. It's gathering huge momentum and we've jumped on for the ride.
